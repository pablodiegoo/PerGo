Esquema do Banco de Dados (Proposta)

CREATE TABLE user_action_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    workspace_id UUID NOT NULL REFERENCES workspaces(id),
    
    -- Identificação Polimórfica do Agente
    actor_type VARCHAR(50) NOT NULL, -- 'user', 'api_key', 'system'
    actor_id VARCHAR(255) NOT NULL,   -- email do usuário ou api_key.id
    actor_name VARCHAR(255) NOT NULL, -- nome cacheado da key / email
    
    -- Detalhes da Ação
    action VARCHAR(100) NOT NULL,    -- ex: 'campaign.create', 'message.send'
    source VARCHAR(50) NOT NULL,      -- 'dashboard' ou 'api'
    
    -- Contexto de Acesso
    ip_address VARCHAR(45),
    user_agent TEXT,
    metadata JSONB,                  -- Parâmetros, IDs de alvos, etc.
    
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Índice composto otimizado para paginação e ordenação por tenant
CREATE INDEX idx_user_action_logs_workspace_date 
ON user_action_logs(workspace_id, created_at DESC);

Fluxo de Captura em Go

Para garantir a captura sem impactar o tempo de resposta do cliente, a gravação de logs deve utilizar a mesma infraestrutura assíncrona do audit.Writer existente.

Dashboard (Echo Handlers)

Nos handlers de renderização/postagem do admin (ex: criar campanha, deletar conexão), o identificador do operador atual (email de login ou ID do usuário da sessão) é extraído do contexto e enviado ao gravador.

API Publica (Echo Middleware)

Um middleware de auditoria global é ativado em todas as rotas da API (/api/v1/*). Ele extrai o IP, User-Agent, ID da chave de API verificada e registra a ação de forma assíncrona com os metadados da requisição.
