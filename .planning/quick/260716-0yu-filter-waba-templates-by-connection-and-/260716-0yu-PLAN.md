# Plan: Filter WABA templates by connection and add phone keywords like fone to CSV parser

We need to scope WABA templates to their specific connection instead of workspace-wide to prevent conflicts and mixed templates, and add "fone" to the CSV parser keywords so that Brazilian mailing lists are parsed correctly.

## Tasks

### 1. Database Migration for WABA Connection Scoping
- Create a migration `017_add_connection_id_to_waba_templates.sql` to add `connection_id` column to `waba_templates`.
- Drop unique constraint `waba_templates_workspace_id_name_language_key` and add `waba_templates_connection_id_name_language_key UNIQUE (connection_id, name, language)`.

### 2. Update WABATemplate Repository and Struct
- Update `WABATemplate` struct in `internal/repository/waba_template.go` to include `ConnectionID` field.
- Update repository queries (`Create`, `Upsert`, `GetByID`, `GetByNameAndLanguage`, `ListByWorkspace`, `Delete`) to handle `connection_id`.
- Add `ListByConnection` query to repository.

### 3. Update Device Sync Handler and Campaign Connection Scoping
- Pre-generate `Connection.ID` in `DeviceHandler.Create` before calling `syncTemplatesFromMeta` so we can pass it down.
- Update `syncTemplatesFromMeta` to save templates with the connection ID.
- Update `campaigns.templ` template selection to filter templates based on connection.
- Update `Campaign` entity and worker to utilize and pass `ConnectionID` or `SenderIdentity` for WABA.

### 4. Add Phone Keywords to CSV Parser
- Update `phoneKeywords` in `internal/api/handler/admin/campaign.go` to include `fone`, `tel`, `contato`, `contact`.
