# Plan: Implement Connections List and Inline Creation in Campaigns Form

## Objective
Enhance the campaigns creation form so that connection selection lists the workspace's configured connections (rather than a hardcoded list of channels), automatically maps their channel types, and includes an "+ Adicionar nova conexão..." option at the end. When selected, this option opens the connection pairing modal, and once the connection is successfully created, the campaigns page form reloads automatically to list the new connection.

## Tasks
1. **Task 1: Update Campaigns Page Layout ([campaigns.templ](file:///home/pablo/Coding/OmniGo/templates/pages/campaigns.templ))**
   - List the active connections `connections` in the `#channel` select element, using the connection's channel as the value and showing the name + channel type in brackets.
   - Include an option `+ Adicionar nova conexão...` with `value="add_new_connection"`.
   - Implement `handleConnectionChange()` in javascript to intercept `add_new_connection` selections, load the `/admin/devices/pair-form` modal into `#modal-container` via `htmx.ajax`, and reset the select value.
   - Add `<div id="modal-container"></div>` at the end of `CampaignsContent`.
   - Add `hx-get` and `hx-trigger="connection-created from:body"` attributes to `#campaign-form-area` to automatically reload the campaign form when a new connection is created.

2. **Task 2: Return Trigger Header on Connection Creation ([device.go](file:///home/pablo/Coding/OmniGo/internal/api/handler/admin/device.go))**
   - Update the `Create` handler in `device.go` to inspect the `HX-Current-URL` header.
   - If the request originated from a campaigns page (contains `/campaigns`), set the `HX-Trigger` header to `connection-created` and return `http.StatusNoContent` (`c.NoContent(200)`).

3. **Task 3: Support Inline Pairing Completion in Modal ([devices.templ](file:///home/pablo/Coding/OmniGo/templates/pages/devices.templ))**
   - Update the pairing success/error case buttons in the QR component in `devices.templ`.
   - Replace the static `hx-get="/admin/devices"` with inline JavaScript in `onclick` that closes the modal and either reloads the connections table (if `#connections-table-container` is present) or fires the `connection-created` custom event on body (if `#campaign-create-form` is present).

4. **Task 4: Verification and compilation**
   - Run `templ generate ./...` and `go test ./...` to verify everything compiles and passes.
