# Phase 25: Implement JSON-to-Protobuf mapping for rich interactive messages (hybrid approach)

<threat_model>
- **ASVS Level:** 1
- **Block On:** high
- **Description:** 
  - **Injection (High):** Processing raw JSON for `channel_overrides` poses an injection risk if passed directly to unsanitized sinks. For WABA (`whatsapp_cloud`), passing raw JSON directly as an HTTP body payload requires ensuring proper content types are maintained. For `whatsmeow`, decoding using `protojson.Unmarshal` safely mitigates malformed payloads but requires robust error handling.
  - **Denial of Service (Medium):** Complex interactive payloads or deep nested structures in overrides might cause excessive CPU consumption during mapping or `protojson` parsing. Downstream components should bound the size of degraded plaintext fallback. Existing Echo layer HTTP body size limits offer primary mitigation.
</threat_model>

## 1. Domain Schema & Validation Extensions
- **File:** `internal/domain/message.go`
  - Define unified structures: `Interactive` (with `Type`, `Header`, `Body`, `Footer`, `Action`) and sub-structures like `Action` and `Button`.
  - Extend `CreateMessageRequest` and `QueueMessage` structs to include:
    - `Interactive *Interactive`
    - `ChannelOverrides map[string]json.RawMessage`
    - `FallbackBehavior string`
  - Update `ValidateMessage()` to ensure `FallbackBehavior` is either empty, `"degrade"`, or `"fail"`. Ensure structural integrity of `Interactive` without enforcing channel-specific limits (D-01).
- **File:** `internal/domain/message_test.go`
  - Add unit tests for `FallbackBehavior` and structural validation logic, verifying that limits (e.g. 4+ buttons) are correctly allowed to pass the gateway layer.

## 2. Ingestion & Orchestration Propagation
- **File:** `internal/outbound/processor.go`
  - Update `Processor.Ingest` to map `Interactive`, `ChannelOverrides`, and `FallbackBehavior` from `req` (`CreateMessageRequest`) to `qMsg` (`QueueMessage`) before publishing to NATS.
- **File:** `internal/channel/dispatcher.go`
  - Update `MessagePayload` to include `Interactive`, `ChannelOverrides`, and `FallbackBehavior`.
- **File:** `internal/platform/queue/orchestrator.go`
  - Update `dispatchToChannel` to map the new fields from `domain.QueueMessage` to `channel.MessagePayload` during dispatcher delegation.

## 3. WABA (`whatsapp_cloud`) Adapter Mapping
- **File:** `internal/channel/whatsapp/waba.go`
  - Implement full override handling: if `channel_overrides.whatsapp_cloud` is present, inject the raw JSON directly as the WABA API HTTP payload (D-02).
  - Implement unified mapping: if no override is present and `Interactive` exists, map it into Meta's standard `"type": "interactive"` JSON structure.
- **File:** `internal/channel/whatsapp/waba_test.go`
  - Add tests to ensure unified schema maps to valid Meta interactive JSON, and that `channel_overrides.whatsapp_cloud` completely replaces the payload.

## 4. WhatsApp Web (`whatsmeow`) Adapter Mapping & Fallbacks
- **File:** `internal/channel/whatsapp/adapter.go`
  - Implement override handling: if `channel_overrides.whatsapp` is present, use `protojson.Unmarshal` to parse the raw JSON directly into a `waE2E.Message`.
  - Implement unified mapping: convert `Interactive` into the appropriate `waE2E.Message` structures (`waE2E.ButtonsMessage`, `waE2E.ListMessage`, etc.).
  - Implement degradation & fallback (D-03): check native limits (e.g., max 3 buttons). 
    - If violated and `FallbackBehavior == "fail"`, return a `TerminalError`.
    - If `FallbackBehavior == "degrade"`, elegantly degrade the interactive elements into a plain-text payload (e.g., text body + numbered list) and clear the interactive components before dispatch.
- **File:** `internal/channel/whatsapp/adapter_test.go`
  - Write test cases asserting `protojson` unmarshaling for overrides.
  - Verify unified `Interactive` schema translates accurately into `waE2E.Message` variants.
  - Assert that `fallback_behavior == "degrade"` correctly transforms payloads when native limits are exceeded, and `"fail"` emits `TerminalError`.
