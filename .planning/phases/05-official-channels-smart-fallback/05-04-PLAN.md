<task type="auto" tdd="true">
  <name>Task 3: Worker fallback dispatch loop implementation</name>
  <read_first>
    - internal/platform/queue/worker.go
    - internal/repository/dispatch.go
    - .planning/phases/05-official-channels-smart-fallback/05-RESEARCH.md
  </read_first>
  <acceptance_criteria>
    - `Worker.processMessage` is refactored to fetch or create the dispatch state in `message_dispatches` table using the trace ID.
    - If dispatch state has status `"sent"`, the worker logs duplicate skip and calls `Ack` on the NATS message immediately.
    - The worker executes an iterative fallback loop starting at the dispatch state's `current_channel` and `fallback_index`.
    - On terminal error, if fallback channels remain, the worker updates status to `"sending"`, increments `fallback_index`, and loops to the next channel.
    - If terminal error occurs and fallbacks are exhausted, status is updated to `"failed"` and NATS message is `Ack`ed (ending queue retry).
    - On transient errors, status is updated to `"failed_transient"` and NATS message is `Nak`ed with backoff.
  </acceptance_criteria>
  <action>
    - Modify `internal/platform/queue/worker.go` to inject `MessageDispatchRepository` and implement the stateful fallback dispatch loop in `processMessage` and `dispatch`.
    - Modify `cmd/omnigo/main.go` to pass `MessageDispatchRepository` to `NewWorker`.
  </action>
</task>
