# Plan 2: Split-Pane Page, Sidebar, and Conversation List

This plan establishes the visual shell of the Conversational Inbox. It adds the sidebar link, lays out the split-pane HTML templates, registers HTTP handlers in Echo, and implements dynamic conversation list rendering with cookie-based unread tracking and 5s HTMX polling.

## Tasks

<task id="migration_inbox_read_status">
  <read_first>
    - internal/repository/recipient_session.go
  </read_first>
  <action>
    Create a new goose migration file at `internal/platform/postgres/migrations/014_inbox_read_status.sql` to add a `last_read_at` TIMESTAMPTZ column to the `recipient_sessions` table:
    ```sql
    -- +goose Up
    ALTER TABLE recipient_sessions ADD COLUMN last_read_at TIMESTAMPTZ;
    
    -- +goose Down
    ALTER TABLE recipient_sessions DROP COLUMN last_read_at;
    ```
    Implement the `UpdateLastReadAt(ctx context.Context, workspaceID uuid.UUID, recipientPhone, channel, recipientIdentity string, lastReadAt time.Time) error` method on `RecipientSessionRepository` (in `internal/repository/recipient_session.go`) to update the `last_read_at` column for a specific instance's session.
  </action>
  <acceptance_criteria>
    - The new migration file is created under the correct path.
    - Database migration runs successfully.
    - `RecipientSessionRepository` has the method to update `last_read_at`.
  </acceptance_criteria>
</task>

<task id="register_inbox_routes">
  <read_first>
    - cmd/pergo/main.go
  </read_first>
  <action>
    Modify `cmd/pergo/main.go` to register the new admin handlers on the protected admin router group:
    ```go
    // GET /admin/inbox - renders split-pane layout and conversation list
    adminGroup.GET("/inbox", inboxHandler.View)
    // GET /admin/inbox/conversations/poll - returns conversation list HTML fragment for polling
    adminGroup.GET("/inbox/conversations/poll", inboxHandler.PollConversations)
    // GET /admin/inbox/chat - returns chat history panel for a contact
    adminGroup.GET("/inbox/chat", inboxHandler.ChatPanel)
    // GET /admin/inbox/messages - returns new messages for chat polling
    adminGroup.GET("/inbox/messages", inboxHandler.PollMessages)
    // POST /admin/inbox/send - enqueues outbound replies
    adminGroup.POST("/inbox/send", inboxHandler.SendMessage)
    ```
  </action>
  <acceptance_criteria>
    - Router setup compiles without warnings.
    - All routes map to the appropriate `inboxHandler` methods.
  </acceptance_criteria>
</task>
