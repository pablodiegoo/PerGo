# Phase 26: Implement Telegram Inline Keyboards and Forum Threads mapping

## Goal
Map Telegram inline keyboards and forum threads into the unified payload schema, handling inbound button clicks (callback queries) and correctly routing forum thread context.

## requirements
- TELE-01: Support Telegram Inline Keyboards and Forum Threads routing via the generic `POST /messages` payload schema.

## must_haves
- Interactive message elements (buttons) must map to Telegram's Inline Keyboard Markup in outbound requests.
- Inbound callback queries must be successfully parsed into `InboundEvent.Interactive` and acknowledge the query via Telegram's `answerCallbackQuery` API.
- `thread_id` metadata must be properly mapped inbound and outbound between Telegram's `message_thread_id` and the PerGo unified schema.
- The platform inbound processor must preserve and publish the new `Interactive` structure downstream.
