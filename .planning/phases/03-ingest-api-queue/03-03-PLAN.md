<objective>
Rate limiting, backpressure, retry with backoff, TTL enforcement, and delivery deduplication.

Purpose: Production-hardens the message ingestion path. After this plan, the system handles load gracefully — rate limits per workspace, enforces queue depth, retries failures with exponential backoff, drops expired messages, and prevents duplicate delivery. The ingest API is production-ready for Phase 4 channel dispatch.

**Deferred: Staggered dispatch (1-3s random delay)** — The ROADMAP success criterion #5 specifies "per-session rate limiting applies staggered dispatch (1-3s random delay) for unofficial WhatsApp channels." This is a channel-dispatch concern (WhatsApp Web ban-risk mitigation), not an ingest-API concern. The token-bucket rate limiter in this plan controls throughput at the API layer; the staggered delay will be implemented in Phase 4's WhatsApp Web worker dispatch loop where it directly mitigates ban risk. The `Dispatcher` interface defined in Phase 3 Plan 02 provides the injection point for this behavior.

Output: Rate limiter middleware, updated worker with retry/TTL/dedup, updated handler with queue depth check.
</objective>

| HTTP Client → Rate Limiter | Caller may bypass rate limit by spoofing workspace_id — mitigated by auth middleware injecting trusted workspace_id |
| JetStream → Worker Retry | Retried messages may be poisoned — exponential backoff with max retries prevents infinite loops |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-07 | Denial of Service | Rate limit bypass | high | mitigate | workspace_id from auth context (trusted), not from payload; rate limiter middleware runs before handler |
| T-03-08 | Denial of Service | Queue depth bypass | high | mitigate | Queue depth checked BEFORE publish with atomic counter; workspace_id from trusted auth context |
| T-03-09 | Denial of Service | Retry storm | medium | mitigate | MaxRetries=5 with exponential backoff capped at 60s; terminal failures acked and logged |
