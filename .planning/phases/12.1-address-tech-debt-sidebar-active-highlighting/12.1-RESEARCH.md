# Phase 12.1 Research: Sidebar Active Highlighting and Settings Accordion

## Overview
This document contains the research on fixing the sidebar active styling highlighting and Settings accordion expansion bugs for workspace-scoped campaigns and webhooks.

## Key Findings

### 1. Files & Components Involved
- `templates/layout/sidebar.templ` (and generated `templates/layout/sidebar_templ.go`) - Left sidebar navigation component and path-matching helpers.
- `templates/pages/campaigns.templ` - Layout container for workspace-scoped Campaigns pages.
- `cmd/pergo/main.go` - Route handlers and path-matching middleware setting `"active_path"`.

### 2. Highlighting & Active State Bugs
- **Campaigns Page Highlighting**: Navigating to `/admin/campaigns` redirects to `/admin/workspaces/:workspace_id/campaigns`. The middleware sets `active_path` to `/admin/workspaces/:workspace_id/campaigns`. Since the current `isTopLevelActive` function only checks if the path begins with `/admin/campaigns`, it returns `false`, causing the "Campanhas" link to lose active highlighting.
- **Settings Accordion Expansion**: Since the workspace campaigns URL begins with `/admin/workspaces`, `isSettingsActive` evaluates to `true` and incorrectly expands the Settings accordion.
- **Webhooks Page Highlighting**: Visiting the workspace-scoped webhooks page (`/admin/workspaces/:workspace_id/webhooks`) expands the Settings accordion but highlights the "Workspace" submenu item incorrectly because the path starts with `/admin/workspaces/` instead of `/admin/webhooks/`, leaving the "Webhooks" submenu item unhighlighted.

### 3. Proposed Fixes
We can modify the route-matching helper functions in `templates/layout/sidebar.templ`:

1. **Modify `isSettingsActive`** to exclude paths containing `/campaigns`:
```go
func isSettingsActive(activePath string) bool {
    if strings.Contains(activePath, "/campaigns") {
        return false
    }
    prefixes := []string{
        "/admin/connections",
        "/admin/devices",
        "/admin/logs",
        "/admin/audit",
        "/admin/workspace",
        "/admin/workspaces",
        "/admin/webhooks",
        "/admin/telemetry",
    }
    for _, p := range prefixes {
        if strings.HasPrefix(activePath, p) {
            return true
        }
    }
    return false
}
```

2. **Modify `isTopLevelActive`** to support workspace-scoped campaigns URLs:
```go
func isTopLevelActive(activePath, target string) bool {
    if target == "/admin/" {
        return activePath == "/admin" || activePath == "/admin/"
    }
    if target == "/admin/inbox" {
        return strings.HasPrefix(activePath, "/admin/inbox")
    }
    if target == "/admin/campaigns" {
        return strings.HasPrefix(activePath, "/admin/campaigns") || strings.Contains(activePath, "/campaigns")
    }
    return false
}
```

3. **Modify `isSubmenuActive`** to prevent overlapping of the `"Workspace"` submenu and correctly target `"Webhooks"` pages:
```go
func isSubmenuActive(activePath, target string) bool {
    if target == "/admin/connections" {
        return strings.HasPrefix(activePath, "/admin/connections") || strings.HasPrefix(activePath, "/admin/devices")
    }
    if target == "/admin/logs" {
        return strings.HasPrefix(activePath, "/admin/logs") || strings.HasPrefix(activePath, "/admin/audit")
    }
    if target == "/admin/workspace" {
        return (strings.HasPrefix(activePath, "/admin/workspace") || strings.HasPrefix(activePath, "/admin/workspaces")) &&
            !strings.Contains(activePath, "/webhooks") &&
            !strings.Contains(activePath, "/campaigns")
    }
    if target == "/admin/webhooks" {
        return strings.HasPrefix(activePath, "/admin/webhooks") || strings.Contains(activePath, "/webhooks")
    }
    if target == "/admin/telemetry" {
        return strings.HasPrefix(activePath, "/admin/telemetry")
    }
    return false
}
```
