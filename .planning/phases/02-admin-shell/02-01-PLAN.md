<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Admin Panel | Unauthenticated users must not access admin routes; session cookie is the auth boundary |
| Login form → Session | Password must not be logged; session cookie must be signed to prevent forgery |
| HTMX fragment → Full page | Fragment responses must not leak data intended only for full pages |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Tampering | Session cookie | high | mitigate | Signed cookie with HMAC; HttpOnly + SameSite=Lax flags |
| T-02-02 | Elevation of Privilege | Admin routes | high | mitigate | Session middleware on /admin/* group; redirect to login if unauthenticated |
| T-02-03 | Information Disclosure | Login errors | low | accept | Generic "invalid password" message; no username enumeration (single operator) |
| T-02-04 | Tampering | CSS injection via templ | low | accept | Templ auto-escapes by default; never use templ.Raw() on user data |
| T-02-SC | Tampering | npm/pip/cargo installs | high | mitigate | Package Legitimacy Audit passed (templ OK, HTMX OK) — no blocking checkpoint needed |
</threat_model>
