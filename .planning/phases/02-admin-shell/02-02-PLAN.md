<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Form submission → Server | Workspace names and API key names must be validated; XSS via name fields prevented by templ auto-escaping |
| Delete action → Database | Workspace deletion cascades to API keys; confirmation modal prevents accidental deletion |
| API key plaintext → Browser | Plaintext key shown only once on generation; never stored or returned in list views |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-05 | Tampering | Workspace name | low | accept | Templ auto-escapes; no raw HTML in names |
| T-02-06 | Elevation of Privilege | Workspace deletion | medium | mitigate | Confirmation modal required before delete; session auth enforced |
| T-02-07 | Information Disclosure | API key plaintext | high | mitigate | Key shown once on generate; never returned in list/detail views; repository does not expose plaintext after creation |
| T-02-08 | Denial of Service | Mass workspace creation | low | accept | Rate limiting deferred; single-operator model limits blast radius |
</threat_model>
