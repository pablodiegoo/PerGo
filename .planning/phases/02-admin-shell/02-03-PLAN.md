<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filter input → SQL query | Filter parameters must be parameterized (not concatenated) to prevent SQL injection |
| CSV export → Download | CSV file must not contain sensitive payload data beyond what audit_logs stores |
| Pagination → Database | Large result sets must not cause OOM; LIMIT/OFFSET enforced at query level |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-09 | Tampering | SQL injection via filters | critical | mitigate | All filter values passed as parameterized queries ($1, $2, etc.); never string-concatenate into SQL |
| T-02-10 | Information Disclosure | Audit log payload | low | accept | Payload is JSONB set by application; no PII scrubbing needed at this layer |
| T-02-11 | Denial of Service | Unbounded CSV export | medium | mitigate | CSV export uses same filter as list view; no "export all without filter" endpoint |
| T-02-12 | Information Disclosure | Trace ID exposure | low | accept | Trace IDs are internal identifiers; exposure to authenticated operator is expected |
</threat_model>
