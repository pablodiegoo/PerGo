# Phase 23, Plan 01: Stateful Handoff Routing â€” Wave 1

<objective>
Introduce the core backend structures and automated behavior for stateful handoff routing. This includes database migrations, domain/repository updates, Typebot interception logic, lazy evaluated inactivity cooldown resets, messaging verbs extension for `pause_bot`, and Chatwoot agent reply auto-pausing.
</objective>

<must_haves>
- D-01: Contacts schema must persist `bot_active` (boolean, DEFAULT true) and `bot_paused_at` (timestamp, nullable) columns, mapped inside Go's `domain.Contact` and `repository.ContactRepository`.
- D-02: `TypebotForwarder.SyncInboundMessage` must check `contact.BotActive` and skip API calls and outbound dispatch if set to `false`.
- D-03: Receiving an outgoing, public user reply webhook from Chatwoot must set `bot_active = false` and `bot_paused_at = NOW()`.
- D-05: Webhook verbs execution engine must support a `pause_bot` action with an optional duration parameter (e.g. "2h"). If duration is provided, the reset calculation is adjusted accordingly.
- D-07: Inbound processor must perform a lazy evaluated 12-hour inactivity cooldown reset when parsing new customer messages.
</must_haves>

            contact.BotPausedAt = nil
        }
    }
}
