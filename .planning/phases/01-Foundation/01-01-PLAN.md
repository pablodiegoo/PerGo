       - Request ID middleware (generates unique request ID per request)
       - Return the configured Echo instance

    4. **internal/api/handler/health.go**: HealthHandler struct holding `*pgxpool.Pool` and a NATS connection interface. Two methods:
       - `Healthz(c *echo.Context) error` — always returns 200 with body "ok"
       - `Readyz(c *echo.Context) error` — pings pgx pool, pings NATS connection, returns 200 if both succeed, 503 if either fails
       - Use a simple interface for NATS: `type NATSConn interface { Ping() error }` so the handler does not import nats.go directly

    5. **internal/platform/postgres/pool.go**: 
       - `NewPool(ctx context.Context, dsn string) (*pgxpool.Pool, error)` — creates pgxpool with config from DSN
       - `NewSQLDB(pool *pgxpool.Pool) *sql.DB` — wraps pool via `stdlib.OpenDBFromPool(pool)` for goose and future whatsmeow

    6. **internal/platform/postgres/migrate.go**:
       - Embed migrations directory via `//go:embed migrations/*.sql`
       - `RunMigrations(db *sql.DB) error` — sets goose base FS, dialect "postgres", runs `goose.Up(db, "migrations")`
       - Use the stdlib `*sql.DB` (not pgxpool) for goose compatibility

    7. **migrations/001_create_schema.sql**: Create the four foundational tables:
       - `workspaces`: id (uuid PK), name (text unique not null), created_at (timestamptz default now()), updated_at (timestamptz default now())
       - `api_keys`: id (uuid PK), workspace_id (uuid FK), key_prefix (text not null), key_hash (bytea not null), name (text), revoked_at (timestamptz nullable), key_id (text not null), key_version (int not null default 1), created_at (timestamptz default now())
       - `devices`: id (uuid PK), workspace_id (uuid FK), channel (text not null), device_id (text not null), status (text not null default 'pending'), credentials (bytea), key_id (text), key_version (int default 1), created_at (timestamptz default now()), updated_at (timestamptz default now())
       - `audit_logs`: id (uuid PK), workspace_id (uuid not null), trace_id (text not null), event_type (text not null), payload (jsonb), created_at (timestamptz default now())
       - Add indexes: api_keys(workspace_id), api_keys(key_prefix), devices(workspace_id), audit_logs(trace_id), audit_logs(created_at)
       - Add foreign keys: api_keys.workspace_id -> workspaces(id), devices.workspace_id -> workspaces(id)
       - Note: audit_logs partitioning by created_at will be added in a later migration when the partition function is ready; the base table is created here

    8. **cmd/pergo/main.go**: Composition root that:
       - Reads config from env vars: DATABASE_URL (default postgres://postgres:postgres@localhost:5432/pergo?sslmode=disable), NATS_URL (default nats://localhost:4222), SERVER_PORT (default 8080), DEBUG_PORT (default 6060)
       - Creates pgxpool.Pool via postgres.NewPool()
       - Creates stdlib *sql.DB via postgres.NewSQLDB()
       - Runs goose migrations via postgres.RunMigrations()
       - Connects to NATS via nats.Connect()
       - Creates Echo server via echo.New()
       - Registers health handlers: GET /healthz -> Healthz, GET /readyz -> Readyz
       - Starts Echo server on configured port
       - Handles SIGTERM/SIGINT for graceful shutdown (basic signal handling for now; full shutdown orchestrator in Plan 4)
       - On shutdown: close Echo, close NATS, close pgxpool, close sql.DB

       - `docker-up`: `docker-compose up -d`
       - `docker-down`: `docker-compose down`
