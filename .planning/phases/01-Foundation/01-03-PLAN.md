       - `Writer` interface: `Write(e Event)`, `Close() error`
       - `NewWriter(pool *pgxpool.Pool, bufSize int, workers int) Writer` — creates bounded channel (cap bufSize) and spawns workers
       - `Write(e Event)` — non-blocking send to channel; if channel full, increment drop counter and log warning
       - `Close() error` — closes channel, waits for all workers to drain (sync.WaitGroup), returns nil

    3. **internal/platform/audit/batch.go**:
       - `BatchWriter` struct: ch (chan Event), pool (*pgxpool.Pool), wg (sync.WaitGroup), batchSize (int)
       - `worker()` method — collects events into batch slice; when len(batch) >= batchSize OR channel closed, calls flush()
       - `flush(events []Event)` — acquires pgx connection via pool.Acquire(), uses `conn.Conn().CopyFrom()` to bulk insert into audit_logs table via `pgx.Identifier{"audit_logs"}` and `pgx.CopyFromSlice`
       - Columns for CopyFrom: workspace_id, trace_id, event_type, payload, created_at
       - On flush error: log error, increment drop counter (expvar), do not panic
       - Default batchSize = 100

    4. **internal/platform/postgres/002_partition_audit.sql**:

       - Create initial partition for current month: `SELECT create_monthly_partition(CURRENT_DATE)`
       - Note: The base audit_logs table was created in migration 001. This migration adds partitioning support. In a real deployment, you'd detach the base table and attach partitions. For MVP, create the partition function and first partition. The base table remains for simplicity; partition management will be enhanced in later phases.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → Trace middleware | Trace-ID from untrusted header must be validated |
| Audit writer → PostgreSQL | Audit events cross into persistent storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering | Trace-ID spoofing via header | medium | accept | Trace-ID is for debugging, not security; spoofed IDs harm attacker only (own traces become untraceable) |
| T-03-02 | Denial of Service | Audit channel overflow | medium | mitigate | Bounded channel (cap 5000) with non-blocking write; overflow increments drop counter, does not block handlers |
| T-03-03 | Information Disclosure | Audit payload contains sensitive data | medium | accept | Audit events are operator-controlled; payload schema is defined by the application, not user input |
| T-03-04 | Tampering | Audit log tampering | low | accept | Immutable partitioned table; no UPDATE/DELETE granted to application role; dedup lives upstream |
| T-03-SC | Tampering | Go module dependencies | high | mitigate | All packages are stdlib (crypto, log/slog, expvar) or well-established (pgx, google/uuid) |
</threat_model>
