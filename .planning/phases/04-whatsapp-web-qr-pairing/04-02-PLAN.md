Output: DeviceRepository, SessionRegistry, SessionManager with startup reconnect, migration 003.
</objective>

<task type="auto" tdd="true">
  <name>Task 3: SessionManager with startup reconnect (RED + GREEN)</name>

  <description>
  Build the session lifecycle manager:
  - SessionManager reads active devices from DB on startup
  - Reconnects with semaphore cap (5 concurrent) and jittered backoff (1-5s random between starts)
  - Each device gets its own goroutine running WhatsAppClient.Run(ctx)
  - On LoggedOut event from client: mark device terminal in DB, remove from registry, emit audit event
  - On ClientOutdated event: call SetWAVersion, reconnect
  - Start(ctx) begins reconnection loop
  - Stop() cancels all session contexts, waits for goroutines
  - AddDevice(workspaceID, phone) creates new device entry and starts session
  - RemoveDevice(jid) stops session and marks terminal
  </description>

  <deviations>
  - Manager uses golang.org/x/sync/semaphore for concurrent startup cap
  - Jittered backoff: random sleep 1-5s between device starts (crypto/rand)
  - Manager owns the WhatsAppClient lifecycle — adapter just references the client
  - For MVP, AddDevice returns a QR code channel for pairing flow (Plan 03)
  </deviations>

  <files_modified>
    - internal/session/manager.go
    - internal/session/manager_test.go
  </files_modified>

  <success_criteria>
    - [ ] Startup reconnects active devices with semaphore cap
    - [ ] LoggedOut marks device terminal and removes from registry
    - [ ] ClientOutdated triggers version refresh and reconnect
    - [ ] Stop cancels all sessions cleanly
  </success_criteria>
