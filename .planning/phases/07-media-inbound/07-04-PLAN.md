  <acceptance_criteria>
- WebhookWorker retrieves workspace pii_opt_in value from database.
- If pii_opt_in is false:
  - Hashes sender phone number (from field) using SHA-256.
  - Removes location and contact data arrays from payload.
- HMAC signs the final payload using SignPayload.
- Dispatches event to configured webhook URL.
- Retries on transient failure up to 10 times. Writes to DLQ on permanent failure.
- Inbound webhook forwarding tests verify redaction.
  </acceptance_criteria>
  <action>
In internal/platform/queue/webhook_worker.go, fetch the workspace record. Check the pii_opt_in status. If pii_opt_in is false, calculate SHA-256 hash of InboundEventPayload.Data.From and set InboundEventPayload.Data.Location to nil and InboundEventPayload.Data.Contacts to nil. Marshal the modified payload, compute HMAC signature using SignPayload, and POST to the webhook URL. Move to DLQ via dlqRepo.InsertDLQ if delivery fails terminally or exhausts 10 retries. Update tests in webhook_worker_test.go.
  </action>
  <verify>
    <automated>go test -run TestWebhookWorker_Inbound ./internal/platform/queue -v -count=1</automated>
  </verify>
