## Phase Boundary

This phase implements the remediation plan from the Phase 10 Evaluation Review to resolve the identified BLOCKERs and WARNINGs:
1. Setup GitHub Actions CI/CD to run tests and linter on push and PR.
2. Add security unit tests verifying that `/admin/devices/test/ws` requires authentication.
3. Add a unit test verifying that the decommissioned `/admin/playground` route correctly returns `404 Not Found`.
4. Implement a full HTTP handler test for template sending via NATS (verifying form parsing and publishing).
5. Integrate `testcontainers-go` for database integration tests to ensure tests run reliably and do not skip silently.
</domain>

<decisions>
## Implementation Decisions

### 1. CI/CD Workflow
- A `.github/workflows/ci.yml` will be added to run tests and linters. It will spin up Postgres and NATS services.

### 2. WebSocket Security
- Test `TestDeviceHandler_WS_RequiresAuth` must attempt to switch protocol without an active session cookie and verify it is rejected.

### 3. Playground Decommission Verification
- Test `TestPlaygroundRouteDecommissioned` must query `/admin/playground` and verify `404 Not Found`.

### 4. Integration Test Coverage
- Add `TestInboxHandler_NewMessageSend_HTTP` to test the actual Echo handler context, parser, and publisher invocation.

### 5. Deterministic DB Integration Tests
- Replace the port-fallback database initialization logic in handler tests with a clean `testcontainers-go` container launcher in `TestMain`.
</decisions>
