  <files>internal/domain/campaign.go, internal/domain/campaign_test.go</files>
  <read_first>internal/domain/campaign.go</read_first>
  <action>
    Implement logic helpers and unit tests in `internal/domain/campaign.go` and `internal/domain/campaign_test.go`:
    1. Delimiter Sniffer: `SniffDelimiter(firstLine string) rune` checks frequencies of commas, semicolons, and tabs to auto-detect the delimiter, defaulting to `,`.
    2. Phone Sanitizer: `SanitizePhone(phone string) (string, bool)` strips all non-numeric characters, verifying that length is between 10 and 15 digits (E.164-like range). Returns cleaned string and validity.
    3. Variable Interpolator: `ResolveVariables(input string, row map[string]string) string` replaces regex placeholders `\{\{(.+?)\}\}` case-insensitively with values from the row map. If a placeholder corresponds to a header index like `{{0}}`, it looks up 0-based key in the row map. Keeps raw bracket placeholders if column names do not exist.
    4. Duration Estimation: `CalculateDuration(totalValid, batchSize, delaySeconds int) int` calculates total estimated duration in seconds: `ceil(totalValid / batchSize) * delaySeconds` (assuming uniform mean jitter is 0).
    5. Unit Tests: Create `internal/domain/campaign_test.go` covering all the functions above with robust edge cases.
  </action>

    Configure NATS JetStream streams, consumers, and implement the sequential worker processing loop:
    1. Open `internal/platform/queue/jetstream.go` and add:
       - `EnsureCampaignStream(ctx, nc)` setting up `CAMPAIGNS` stream with subject `campaigns.>` and WorkQueuePolicy retention.
       - `EnsureCampaignConsumer(ctx, stream, consumerName)` setting up durable consumer with `MaxAckPending: 1` to guarantee sequential batch delivery across replicas.
    2. Create `internal/platform/queue/campaign_worker.go` that:
       - Implements a background worker loop subscribing to the campaign consumer.
       - Decodes JSON batch tasks: `{"campaign_id": "UUID", "workspace_id": "UUID", "batch_index": 1, "total_batches": 2, "recipients": [{"to": "5511999998888", "variables": {"nome": "João"}}], "delay_seconds": 5}`.
       - Performs database check: Queries campaign status. If state is `cancelled`, ACKs the NATS message immediately and skips batch processing.
       - For each recipient in the batch, formats messages using `ResolveVariables`, saves dispatch record to `message_dispatches` (outbound logs) via `GetOrCreateDispatch` with Trace ID `campaign_${campaign_id}_${recipient}` and campaign metadata, and publishes the `QueueMessage` to NATS `MESSAGES` outbound stream.
       - Calculates sleep delay: configures time sleep `delay_seconds` + uniform random jitter in `[-0.5s, +0.5s]` range, applying it before ACK-ing the batch.
       - If `batch_index == total_batches`, updates campaign status in database to `completed`.
    3. Update `cmd/pergo/main.go` to initialize the campaigns stream, consumer, and start/stop the `CampaignWorker` under the server's lifecycle orchestration.
    4. Write integration tests in `internal/platform/queue/campaign_worker_test.go` checking sequential delivery, cancellation handling, and log enrichment.
  </action>
  <verify>
    Run worker integration tests: `go test -v ./internal/platform/queue/...`
  </verify>
  <acceptance_criteria>

    - campaigns consumer has `MaxAckPending` set to 1.
    - Worker checks campaign status from DB and stops processing if cancelled.
    - Worker applies `delay_seconds` + jitter sleep dynamically.
    - Worker sets campaign to `completed` on the final batch.
    - `go test -run TestCampaignWorker` passes successfully.
  </acceptance_criteria>
