<task type="auto">
  <name>Task 1: Campaign Echo Admin Handler and Route Registrations</name>
  <files>internal/api/handler/admin/campaign.go, internal/api/handler/admin/campaign_test.go, cmd/pergo/main.go</files>
  <read_first>internal/api/handler/admin/waba_template.go, internal/api/handler/admin/telemetry.go, cmd/pergo/main.go</read_first>
  <action>
    Create Echo handlers and configure routing for campaigns:
    1. Create `internal/api/handler/admin/campaign.go` implementing:
       - `ListCampaigns`: handles GET `/admin/campaigns`, listing campaigns in the active workspace and rendering templates.
       - `NewCampaignForm`: handles GET `/admin/campaigns/new` to show template matching options.
       - `UploadCSVPreview`: handles POST `/admin/campaigns/upload` when a CSV is uploaded. Sniffs delimiter, parses the first few lines for preview table, sanitizes all numbers (filtering E.164 length), deduplicates, calculates summary metrics (total, valid, duplicate, invalid), saves skipped rows to campaign DB array, and returns HTML preview segment.
       - `CreateCampaign`: handles POST `/admin/campaigns` saving the campaign metadata, configuration, valid recipients array, and skipped rows list to database, setting status to `draft` or `scheduled`.
       - `DownloadSkippedCSV`: handles GET `/admin/campaigns/:id/skipped/download` dynamically generating and serving a CSV download of skipped rows containing column headers: `Line Number`, `Raw Input`, `Reason`.
       - `StartCampaign`: handles POST `/admin/campaigns/:id/start`. Slices recipients list into batches based on configured batch size, publishes JSON batch tasks to NATS JetStream `campaigns.batches`, and updates DB status to `sending`.
       - `CancelCampaign`: handles POST `/admin/campaigns/:id/cancel` updating campaign status to `cancelled` using HTMX fragment response.
    2. Register these HTTP handlers in `cmd/pergo/main.go` under `adminGroup` router group.
    3. Create `internal/api/handler/admin/campaign_test.go` containing HTTP handler route tests utilizing standard httptest and mock recorders.
  </action>
  <verify>
    Run handler tests: `go test -v ./internal/api/handler/admin/...`
  </verify>
  <acceptance_criteria>
    - Handlers properly process files uploaded via `MultipartForm`.
    - CSV parser returns skipped rows details in JSON/struct format.
    - Start campaign publishes correct batch JSON structures to NATS.
    - Cancel campaign executes state changes safely using transactional locks.
    - `go test -run TestCampaignAPI` runs and succeeds.
  </acceptance_criteria>
  <done>HTTP Echo handlers and routing established for campaigns, verified by automated unit tests.</done>
