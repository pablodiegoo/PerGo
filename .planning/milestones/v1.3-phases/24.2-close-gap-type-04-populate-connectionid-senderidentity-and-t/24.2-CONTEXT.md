## Phase Boundary

Close implementation gap TYPE-04 by populating ConnectionID, SenderIdentity, and TraceID in the TypebotForwarder queue message before publishing it to NATS.

</domain>

<decisions>
## Implementation Decisions

### Message Population
- **D-01:** `ConnectionID` should be populated directly from `event.ConnectionID`.
- **D-02:** `SenderIdentity` should be populated from `event.To` (since the bot is the sender of the outbound message, its identity is who the customer sent the inbound message *to*).
- **D-03:** `TraceID` should be added to the `domain.QueueMessage` payload using the same generated `traceID` UUID string that is used for the `f.publisher.Publish` call.

### the agent's Discretion
None
