### 2. `internal/integration/typebot/forwarder.go`
**Role:** Integration Mapping / Forwarder Logic
**Data Flow:** Receives an `inbound.InboundEvent`, resolves Typebot configuration, constructs the payload with placeholders (e.g. `[Media Attachment]`, `[Unsupported message...]`) and `pergo_metadata`, and then executes `StartChat` or `ContinueChat`.

**Closest Existing Analog:** Existing `SyncInboundMessage` method within `forwarder.go`.

**Concrete Code Excerpts:**
```go
// Before calling StartChat or ContinueChat, compute:
sessionId := fmt.Sprintf("%s:%s", event.ConnectionID.String(), event.From)

messageText := event.Body

// Placeholder logic for media attachments (D-07, D-05)
if hasMedia(event) {
    messageText = "[Media Attachment]"
} else if isUnsupportedMessage(event) { // Unsupported types (D-09)
    messageText = fmt.Sprintf("[Unsupported message: %s]", getMessageType(event))
}

// Flat JSON string or nested map for metadata (D-01, D-06, D-08)
pergoMetadata := map[string]string{
    "media_url":  event.MediaURL,
    "media_type": event.MediaType,
}

// StartChat Payload Assembly
prefilled := map[string]any{
    "Name":           contact.Name,
    "Phone":          event.From,
    "pergo_metadata": pergoMetadata, // D-01 Nest PerGo's metadata
}

req := StartChatRequest{
    Message:            messageText,
    SessionID:          sessionId,
    PrefilledVariables: prefilled,
}

// ContinueChat Payload Assembly
// Uses `messageText` evaluated above to handle placeholders
messages, contErr = f.client.ContinueChat(ctx, cfg.APIURL, session.TypebotSessionID, activeBot.PublicToken, messageText)
```
