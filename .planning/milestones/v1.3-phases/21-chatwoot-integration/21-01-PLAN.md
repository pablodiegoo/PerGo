# Plan: 21-01 — Chatwoot Schema, Repositories, settings UI and Webhook Stub

This plan initiates Wave 1 of the Chatwoot Integration. It sets up the unified database storage structure for integrations and contact mapping tables, adds the Admin settings controls, and implements the basic authenticated receiver endpoint stub.

  </read_first>
  <action>
    Create a new database migration file 027_create_integrations_and_chatwoot_mappings.sql defining two tables and an index:
    1. 'integrations': fields id (UUID, PK), workspace_id (UUID, FK), name (TEXT), provider (TEXT), active (BOOLEAN), config (BYTEA), key_id (TEXT), key_version (INT), created_at (TIMESTAMPTZ), updated_at (TIMESTAMPTZ), with a UNIQUE constraint on (workspace_id, provider). This implements the single integrations table for config storage (per D-01).
    2. 'chatwoot_mappings': fields workspace_id (UUID, FK), contact_id (UUID, FK), connection_id (UUID, NOT NULL, FK referencing connections on delete cascade), chatwoot_contact_id (BIGINT/INT8), chatwoot_conversation_id (BIGINT/INT8), channel (TEXT), sender_identity (TEXT), created_at (TIMESTAMPTZ), updated_at (TIMESTAMPTZ), with PK on (workspace_id, contact_id, connection_id). This implements the mapping table (per D-03).
    3. An index on 'chatwoot_mappings(chatwoot_conversation_id)' to optimize lookups by Chatwoot conversation ID.
    
    Implement 'IntegrationRepository' in internal/repository/integration.go supporting:
    - Save(ctx, i *Integration): encrypts configurations config JSON envelope using CredentialProvider.Encrypt and AES-256-GCM before writing to the BYTEA config column (per D-02), executing INSERT ... ON CONFLICT (workspace_id, provider) DO UPDATE.
    - GetByProvider(ctx, workspaceID, provider): retrieves and decrypts configuration properties using CredentialProvider.Decrypt.
    
    Implement 'ChatwootMappingRepository' in internal/repository/chatwoot_mapping.go supporting:
    - Upsert(ctx, m *ChatwootMapping): executes INSERT ... ON CONFLICT (workspace_id, contact_id, connection_id) DO UPDATE setting chatwoot_conversation_id = EXCLUDED.chatwoot_conversation_id, chatwoot_contact_id = EXCLUDED.chatwoot_contact_id, sender_identity = EXCLUDED.sender_identity, channel = EXCLUDED.channel, updated_at = EXCLUDED.updated_at.
    - GetByContactAndConnection(ctx, workspaceID, contactID, connectionID): retrieves a single mapping.
    - GetByConversationID(ctx, workspaceID, chatwootConversationID): retrieves a mapping using the workspace ID and the Chatwoot conversation ID.
    - Delete(ctx, workspaceID, contactID, connectionID): deletes a mapping.

    Write unit tests for both repositories in integration_test.go and chatwoot_mapping_test.go.
  </action>
  <verify>
    Run repository unit tests:
    `go test -run TestIntegrationRepository ./internal/repository/...`
    `go test -run TestChatwootMappingRepository ./internal/repository/...`
  </verify>
  <acceptance_criteria>
    - DB migration 027 executes successfully with Goose up and down scripts.
    - integrations table has UNIQUE index on (workspace_id, provider) per D-01.
    - chatwoot_mappings table has PRIMARY KEY on (workspace_id, contact_id, connection_id) and includes connection_id (FK to connections) per D-03.
    - chatwoot_mappings table has an index on chatwoot_conversation_id.
    - IntegrationRepository encrypts plaintext configuration envelope using CredentialProvider.Encrypt before database persistence per D-02.
    - ChatwootMappingRepository stores mapping profiles mapping workspace_id/contact_id/connection_id to chatwoot_contact_id/chatwoot_conversation_id, ensuring conversation ID and other fields are updated on conflict (e.g. chatwoot_conversation_id = EXCLUDED.chatwoot_conversation_id) per D-03.
    - ChatwootMappingRepository provides GetByConversationID isolated by workspace_id and a Delete method using connection_id to remove mappings on 404 response.
    - All repository unit tests pass.
  </acceptance_criteria>
</task>
