## 3. Integration & E2E Flows

All E2E flows are verified:
1. **Multi-Webhook Wildcards & DLQ**: Subscriptions with glob patterns (e.g. `message.*`) receive matching events concurrently. Failures write to the DLQ, where operators can manually retry or delete items via the admin console.
2. **Omnichannel Inbox Merging**: Inbound events auto-resolve to contacts. Operators can search and merge profiles in the inbox UI. The thread updates instantly, displaying merged histories from WhatsApp and Telegram.
3. **Webhook Response Verbs Engine**: Webhooks returning declarative verbs are parsed and executed asynchronously in decoupled goroutines. Replies and forwards post tasks to NATS `messages.outbound` queue, and logs write to workspace audit logs.
4. **WABA Status Indicators**: Outbound messages map `provider_message_id` on dispatch. Meta status callbacks resolve dispatches, update DB status, publish NATS notifications, and render SVG delivery indicators (sent, delivered, read, failed) dynamically in chat bubbles.
